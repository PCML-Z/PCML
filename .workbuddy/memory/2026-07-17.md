## 修复 AI 智能体模型选择 UI 不响应

**问题**: AiAgentPage 中 FilterChip 模型选择（DeepSeek/GPT 切换）点击后 UI 不更新。
**根因**: `switchProvider()` 直接修改 Java 对象字段，Compose 无法感知变化。
**修复**: 新增 `currentProvider` Compose State，switchProvider 末尾更新触发重组。

## AI 智能体系统 v2 全面升级

新增 9 个 Java 类 + 重构 3 个现有类 + UI 完整重写。

### 新增核心类
- `ai/role/AgentRole.java` + `RoleManager.java` — 角色系统（4内置角色 + 自定义持久化）
- `ai/prompt/PromptTemplate.java` — ${variable} 变量替换
- `ai/knowledge/KnowledgeBase.java` + `KnowledgeEntry.java` — TF 关键词检索 RAG
- `ai/tool/ToolPlugin.java` + `ToolRegistry.java` — 可扩展工具插件
- `ai/StreamingAssistant.java` — TokenStream 流式接口

### 重构
- `AiManager.java`: 集成全部子系统，SystemMessageProvider 动态提示词，chatStream 流式回调
- `Assistant.java`: 改为 UserMessage 参数（多模态），移除 @SystemMessage
- `AiConfig.java`: 新增 temperature/maxTokens/streamingEnabled/visionEnabled
- `AiAgentPage.kt`: 角色选择器/流式气泡/图片上传/设置分页（模型/角色/知识库）

### 关键 API 踩坑
- LangChain4j 1.0.0 `SystemMessageProvider` 返回 String 非 SystemMessage
- `dev.langchain4j.rag.Query` 包路径变更 → 用手动 RAG 替代 ContentRetriever
- Kotlin 调 Java `is` 前缀 boolean 需显式 `setXxx()` setter
- Compose 图片解码: SkiaImage.makeFromEncoded + toComposeImageBitmap

## AI 智能体直接介入启动器操作 + 移除表情符号

### LauncherTools（16个 @Tool 方法）
新增 `ai/LauncherTools.java`，接受 LauncherCore 引用，暴露启动器全部核心操作：
- 游戏版本：listGameVersions / listInstalledVersions / installGameVersion / checkIntegrity
- 实例管理：listInstances / createInstance / deleteInstance
- 游戏启动：launchGame（自动查 Java + 构建 LaunchProfile + 异步启动）
- 崩溃分析：analyzeCrashLog（扫描 crash-reports + 原因/建议/恢复操作）
- JVM 配置：setMemory / setJvmArgs / setJavaPath
- Java 运行时：listJavaRuntimes / installJava
- 系统信息：getSystemInfo（CPU/内存/GPU/Java 版本）
- 下载配置：setDownloadMirror（OFFICIAL/BMCLAPI）

### 集成方式
- AiManager 新增 `registerLauncherTools(LauncherTools)` 方法
- LauncherCore 构造函数中 AiManager 创建后自动注册 LauncherTools
- `setStatusCallback` 同时转发到 PmclTools 和 LauncherTools

### 移除表情符号
- AgentRole.icon 从 emoji（🤖📦🔧🎨）改为图标名（smart_toy/inventory_2/build/palette）
- AiAgentPage 新增 `getRoleIcon()` 映射到 Material Icons
- 角色选择器按钮和下拉菜单均使用 Icon 组件替代 Text emoji
