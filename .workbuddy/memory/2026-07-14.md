## 新增 P2P 好友聊天系统（内置功能，非插件）
- 用户要求直接做成 PMCL 功能而非插件，集成到 core + ui 模块
- commit `c561bef`，已 push 到 GitHub

### 架构设计
- 复用现有 MultiplayerManager 的 EasyTier/ConnectX/Terracotta 虚拟网络层
- 好友系统在虚拟 IP 上运行 TCP 聊天 + UDP 对等发现
- 无需引入 libp2p 等重型 P2P 框架

### 核心模块 (core/src/main/java/com/pmcl/core/friend/)
| 类 | 职责 |
|---|---|
| `FriendManager` | 总管理器，协调所有子模块，供 LauncherCore 统一入口 |
| `FriendIdentity` | XXXXX-XXXXX-XXXXX-XXXXX-XXXXX 身份 ID（Base32，排除0/1/O/I） |
| `FriendIdentityManager` | 身份生成/持久化/二维码（ZXing 生成 PNG） |
| `FriendProtocol` | JSON 消息协议（discover/msg/friend_req/friend_ack/status） |
| `FriendChatServer` | TCP 聊天服务器，监听随机端口 |
| `FriendChatClient` | TCP 聊天客户端，异步发送 + 读取线程 |
| `FriendPeerDiscovery` | UDP 端口 25410 广播发现（5s 间隔） |
| `FriendStore` | JSON 持久化：friends.json + messages/<id>.json |

### UI 层变更
- `NavDestination.kt`：新增 Friends 导航项（Icons.Filled.People）
- `App.kt`：注册 FriendPage 路由
- `FriendPage.kt`：好友列表 + 聊天界面 + 二维码弹窗 + 好友请求 + 身份卡片
- `I18n.java`：中/英/日三语标签
- `CommandPalette.kt`：搜索关键词（好友/聊天/friend/chat/QR/二维码）
- `LauncherCore.java`：新增 `friend()` 访问器
- 依赖新增：zxing-core:3.5.3, zxing-javase:3.5.3

### 关键设计决策
- 聊天协议：JSON 行分隔（\n），最大 65536 字节
- 身份分享：`pmcl-friend:XXXXX-XXXXX-...:PlayerName` 协议格式
- 聊天记录上限：每会话 500 条
- 发现广播间隔：5 秒
- 好友数据目录：`~/.pmcl/friend-data/`
