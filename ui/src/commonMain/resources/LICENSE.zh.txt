附录 C:与常见开源许可证的差异说明

  本附录帮助理解本许可证与常见开源许可证的核心差异,仅供辅助理解,不构成
  法律建议。

  C.1 与 MIT 许可证的差异:
      - MIT 允许商业用途,本许可证禁止商业用途(见第 2.4 条);
      - MIT 不要求修改版以相同协议分发,本许可证要求(见第 4.4 条);
      - MIT 不限制命名,本许可证限制 "PMCL" 名称的使用(见第 6.2 条)。

  C.2 与 GNU GPL 的差异:
      - GPL 授予专利许可,本许可证明确不授予(见第 3.1 条);
      - GPL 允许商业用途(含销售),本许可证禁止商业用途(见第 2.4 条);
      - GPL 有更严格的 copyleft 联动要求,本许可证的 copyleft 仅限修改版
        本身,不扩展至仅链接或调用的独立作品(见第 1.7 条除外情形)。

  C.3 与 Apache License 2.0 的差异:
      - Apache 授予专利许可并有专利报复条款,本许可证不授予专利许可
        (见第 3.1 条);
      - Apache 允许商业用途,本许可证禁止商业用途(见第 2.4 条);
      - Apache 不要求修改版以相同协议分发,本许可证要求(见第 4.4 条);
      - Apache 有明确的商标报复条款,本许可证通过第 6 条独立规定商标限制。

  C.4 本许可证的定位:
      本许可证介于宽松型(MIT/Apache)与 copyleft 型(GPL)之间,特点为:
      - 开源可得:源代码必须公开,修改版不得闭源(类 GPL);
      - 非商业性:禁止商业用途,保留商业授权空间(异于 MIT/GPL/Apache);
      - 弱 copyleft:copyleft 仅限修改版本身,不联动独立作品(异于 GPL);
